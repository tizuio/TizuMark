# TizuMark 与 GitHub Markdown 渲染差异对比

> 目的：列出 TizuMark 与 GitHub / Gitee 在 Markdown 语法渲染上的**一致性**与**差异**，
> 覆盖 demo.md 演示的全部语法。本次新增 ` ```math ` / ` ```latex ` / ` ```tex ` 围栏支持后，
> TizuMark 在数学语法层面已与 GitHub 对齐（视觉一致，内部实现见下文「最实质差异」）。
>
> 图例：✅ 两端一致 ｜ ⚠️ 有差异（行为/表现不同） ｜ ❌ 仅一端支持

---

## 〇、总览：最实质的两条差异

| 维度 | GitHub / Gitee | TizuMark | 影响 |
|------|----------------|-----------|------|
| 数学渲染引擎 | **MathJax** | **KaTeX** | 个别高级 LaTeX 命令支持范围不同（详见第七节） |
| 软换行策略 | 固定 GFM 软换行（单 `\n` 不换行） | **可开关**（设置项：软换行开/关） | 同一份文档在 GitHub（关）与 TizuMark（开）下段落换行表现可能不同 |

其余差异多为「语法支持度」与「扩展语法」的差异，见各节。

---

## 一、数学公式渲染（本次重点）

GitHub 官方文档（Writing mathematical expressions）确认：

- 行内：`$...$` 或 `$`...`$`（反引号形式，用于表达式含与 Markdown 冲突的字符）
- 块级：`$$...$$`
- **` ```math ` 围栏**：直接写 LaTeX，**无需 `$$` 包裹**；`math` / `latex` / `tex` 三者同义
- 引擎：**MathJax**
- 美元转义：`\$`

| 写法 | GitHub | TizuMark（本次后） | 一致性 | 说明 |
|------|---------|-------------------|--------|------|
| 行内 `$E=mc^2$` | ✅ | ✅ | ✅ | 两端均渲染 |
| 行内反引号 `$`\sqrt{x}`$`（GitHub 官方） | ✅ | ✅ | ✅ | GitHub 官方推荐写法（避免与 Markdown 冲突）；TizuMark 现已支持，将 `text("$")+code(...)+text("$")` 合并为行内 `$...$` 交 KaTeX 渲染 |
| 行内反引号 `` `$E=mc^2$` ``（用户常用） | 部分支持* | ✅ | ✅ | TizuMark 已支持并把内容解包为 `$...$` 供 KaTeX 渲染 |
| 行内反引号 `` `$$E=mc^2$$` ``（用户常用） | 部分支持* | ✅（归一化为行内） | ⚠️ | TizuMark 会先把 `$$...$$` 归一化为 `$...$`，避免 KaTeX 将其渲染成 display math 导致换行 |
| 块级 `$$...$$` | ✅ | ✅ | ✅ | 两端均渲染为居中块 |
| ` ```math ` 围栏 | ✅ | ✅ | ✅ | **本次新增**。TizuMark 内部将围栏归一化为 `$$` 块再走 KaTeX，视觉与 GitHub 一致 |
| ` ```latex ` 围栏 | ✅ | ✅ | ✅ | 本次新增，`latex` 与 `math` 同义 |
| ` ```tex ` 围栏 | ✅ | ✅ | ✅ | 本次新增，`tex` 与 `math` 同义 |
| 围栏内 `$$` 包裹 | 不需要 | 不需要（但内部自动加） | ✅ | GitHub 文档明确无需 `$$`；TizuMark 归一化时自动补 `$$`，用户写作时同样**不要**写 `$$` |
| 美元转义 `\$` | ✅ | ✅ | ✅ | 均按字面量显示 `$` |
| 引擎差异（高级命令） | MathJax | KaTeX | ⚠️ | 见第七节，少数命令两边结果不同 |

\* GitHub 文档只正式列出 `$`...`$`` 这一种反引号形式；实践中 `` `$...$` `` 与 `` `$$...$$` `` 也常被渲染为数学。TizuMark 选择将后两者统一处理为**行内** `$...$`，保证写在同一行时不会换行。

## 二、文本与段落格式

| 语法 | GitHub | TizuMark | 一致性 | 说明 |
|------|---------|-----------|--------|------|
| 粗体 `**x**` | ✅ | ✅ | ✅ | |
| 斜体 `*x*` / `_x_` | ✅ | ✅ | ✅ | |
| 删除线 `~~x~~` | ✅ | ✅ | ✅ | |
| 行内代码 `` `x` `` | ✅ | ✅ | ✅ | |
| 高亮 `==x==` | ❌ | ✅ | ⚠️ | **GitHub 不支持** `==` 高亮语法；TizuMark 渲染为 `<mark>` |
| 上标 `<sup>` / 下标 `<sub>` | ✅（原生 HTML） | ✅（原生 HTML） | ✅ | 均透传原生 HTML |
| 转义 `\* \# \[` | ✅ | ✅ | ✅ | |
| Emoji 短代码 `:smile:` | ✅ | ✅ | ✅ | TizuMark 由 app 层 `processEmojiShortcodes` 转换，渲染器透传 |
| 软换行（单 `\n`） | 不换行（GFM 默认） | **可开关** | ⚠️ | 见总览；TizuMark 设「开」时单换行即换行，GitHub 始终不换行 |
| 硬换行（两空格 / 反斜杠） | ✅ | ✅ | ✅ | 两端一致 |

## 三、块级结构

### 3.1 标题与锚点

| 项 | GitHub | TizuMark | 一致性 |
|----|---------|-----------|--------|
| 标题 `h1~h6` | ✅ | ✅ | ✅ |
| 标题自动生成 `id` 锚点 | ✅ | ✅（本次新增 `rehypeHeadingIds`，规则对齐 `headingToId`） | ✅ |
| 中文标题锚点 | ✅（slug 保留中文） | ✅（保留中文） | ✅ |
| 重复标题去重 `-2/-3` | ✅ | ✅ | ✅ |
| 目录内点击跳转 | ✅ | ✅（点击滚动到对应标题） | ✅ |

### 3.2 Callout / Alert

| 类型 | GitHub | TizuMark | 一致性 |
|------|---------|-----------|--------|
| NOTE / TIP / IMPORTANT / WARNING / CAUTION | ✅（5 种，`> [!NOTE]` 语法） | ✅（同 5 种） | ✅ 语法一致 |
| INFO 类型 | ❌（GitHub 无，NOTE 含其语义） | ✅（额外支持） | ⚠️ TizuMark 多一种 INFO |
| 标题语言 | 英文固定（"Note" 等） | 中文（"提示"/"注意" 等） | ⚠️ 显示文案不同 |

> 注：GitHub 的 alert 语法 `> [!TYPE]` 与 TizuMark 的 callout 语法**完全一致**，文档可跨平台通用。

### 3.3 列表 / 引用 / 表格

| 语法 | GitHub | TizuMark | 一致性 |
|------|---------|-----------|--------|
| 无序 / 有序列表 + 嵌套 | ✅ | ✅ | ✅ |
| 任务列表 `- [ ]` / `- [x]` | ✅ | ✅ | ✅ |
| 引用块 `>` / 多层 `>>` | ✅ | ✅ | ✅ |

## 七、最实质差异：MathJax vs KaTeX 命令支持范围

GitHub 用 **MathJax**，TizuMark 用 **KaTeX**。两者都支持标准 LaTeX 数学，但：

| 能力 | MathJax（GitHub） | KaTeX（TizuMark） | 影响 |
|------|-------------------|-------------------|------|
| 常用数学命令（`\frac`/`\sum`/`\int`/`\sqrt`/矩阵/`\begin{align}` 等） | ✅ | ✅ | 约 99% 日常公式两边一致 |
| `\label` + `\ref` 交叉引用 | ✅ | ⚠️ 有限支持 | 含交叉引用的论文类公式可能不同 |
| 部分 AMS 扩展命令（`\DeclareMathOperator`、`\xRightarrow` 等） | ✅ | ⚠️ 子集 | 少数命令 KaTeX 不支持，会原样显示 |
| 自定义宏 `\newcommand`（复杂定义） | ✅ | ⚠️ 有限 | 复杂宏定义可能失效 |
| 渲染速度 | 较慢（按需加载） | 更快（静态） | 体验差异，非正确性 |

**实践建议**：若同一份含数学的文档需同时发布到 Gitee/GitHub 与 TizuMark，
优先使用「常用 LaTeX 命令集」，避免依赖 MathJax 独有扩展，可保证两端渲染一致。

## 八、结论

1. **数学语法已全面对齐**：`$...$`、`$$...$$`、` ```math `/`latex`/`tex` 围栏，TizuMark 与 GitHub 写法一致、视觉一致。
2. **数学语法已与 GitHub 完全一致**：GitHub 官方推荐的「行内反引号 `$`...`$`」形式 TizuMark 现已实现（见第 20 节实现记录）；TizuMark 额外支持的 `` `$...$` `` / `` `$$...$$` `` 两种用户常用写法统一按行内渲染，与 GitHub 实践行为一致。
3. **TizuMark 增强项**（GitHub 不支持，文档在 GitHub 会降级为普通文本/代码）：
   `==高亮==`、定义列表、`` [TOC] ``、图片尺寸属性、`file://` 图片、Mermaid 灯箱。
4. **GitHub 独有 / 不同**：callout 的 INFO 类型、callout 标题英文、软换行固定策略。
5. **引擎级差异**：MathJax vs KaTeX 导致少数高级 LaTeX 命令两边结果不同（见第七节）。

> 文档可跨平台通用（GitHub 支持的语法 TizuMark 均支持），仅 TizuMark 的扩展语法在 GitHub 上会降级。
